# Contributing to A.O.A.I

We welcome contributions that strengthen defensive capabilities, privacy safeguards, accessibility, or transparency. Offensive tooling or surveillance features will be rejected.

## Development Workflow
1. Fork the repository and create a feature branch.
2. Run `make bootstrap` to install dependencies.
3. Ensure `make test` and `make lint` pass.
4. Submit a pull request summarizing the change, referencing relevant documentation.

## Code of Conduct
- Respect user sovereignty and privacy.
- No hate speech, discrimination, or harassment.
- Follow responsible disclosure practices for vulnerabilities.

## Security Reporting
Email security@aoai.example.org with encrypted reports (PGP key available on our website).
